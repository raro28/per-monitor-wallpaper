# Toolchain port to TypeScript + esbuild (v2.0.0)

- Date: 2026-06-21
- Status: approved design, pre-implementation
- Target: `per-monitor-wallpaper@ekthor`, GNOME Shell 50
- Prerequisite for: the v2.1.0 preview-preferences feature
  (`2026-06-21-v2.1.0-preview-prefs-design.md`)

## Goal

Re-platform the existing extension from hand-written GJS to a TypeScript + esbuild
codebase with SOLID module decomposition, dev type-checking/linting, a CI-built release
artifact, and a build-free RPM — **with no change to runtime behavior**. This proves the
stack on the real code before the v2.1.0 feature is built on it.

## Acceptance bar

**Behavior parity with 1.0.2.** The built v2.0.0 extension must apply wallpapers
identically: per-monitor desktop backgrounds, overview backgrounds, login seed, rotated
monitors, and live `config.json` reload all behave exactly as 1.0.2. The `mode` field
stays ignored (always `zoom`) — honoring it is v2.1.0, not here.

## Verified on host

- esbuild path proven in `/tmp` with real `esbuild` + `gjs 1.88`: a `--bundle
  --format=esm '--external:gi://*' '--external:resource://*'` build of a TS entry leaves
  `gi://…` and `resource:///org/gnome/shell/…` imports external, bundles local `./lib/*`
  inline, preserves `export default class extends Extension`, and the output runs under
  `gjs -m`.
- Tests proven on host (`node` 23.7 / `npm` 10.9): esbuild a `.ts` test → JS, then
  `node --test` runs green. (`node --test` does not run `.ts` directly here.)
- `@girs/gnome-shell` 50.0.2 exists; its dependency closure includes `@girs/gtk-4.0`,
  `@girs/adw-1`, `@girs/gdk-4.0` (via gtk), `@girs/meta-18`, `@girs/shell-18`,
  `@girs/st-18`, `@girs/clutter-18`, `@girs/gio-2.0`, `@girs/glib-2.0`,
  `@girs/gobject-2.0` — one package covers every gi:// namespace this project imports.
  Linting uses `typescript-eslint` (parser + plugin) on ESLint 9 flat config;
  `eslint-plugin-gjs` is unusable (its 1.0.4 release imports `eslint/lib/ast-utils`,
  removed in ESLint 9, and its ruleset bans ESM/classes/template literals).

Phase-0-locked (exist, exact form produced during implementation, not asserted as
proven): the `tsconfig.json` + `@girs` wiring for `tsc --noEmit`; the one-line
`cp metadata.json dist/` in the build script.

## Scope

In scope:
- Port `src/extension.js` → TypeScript modules (decomposition below). No behavior change.
- `package.json` + esbuild build → `dist/extension.js` (+ `dist/metadata.json`).
- `tsc --noEmit` type-check with `@girs` types; `eslint` (`eslint-plugin-gjs`).
- A test runner for the GI-free `src/lib/` modules.
- `install.sh`: build then install `dist/`.
- GitHub Actions: on tag, build and attach a release tarball of `dist/`.
- rpm-specs: `Source0` → the release asset; bump `Version` 1.0.2 → 2.0.0; metadata
  `version` 1 → 2.

Out of scope:
- Any runtime behavior change, including honoring `mode` (v2.1.0).
- `prefs.js` / the preview UI (v2.1.0).

## Architecture (SOLID decomposition)

`extension.ts` becomes a thin orchestrator that constructs and wires focused modules
(lightweight constructor injection; no DI container). The existing 22-method class splits
by responsibility:

- `src/lib/` — **GI-free, unit-tested** pure logic:
  - `config.ts` — `parseConfig(text): Config` (tolerant: invalid → `{}`),
    `fileForConnector(cfg, connector): string | null` (the current `_idxToFile`
    resolution as a pure function: monitor entry's `file`, else `cfg.default` as a
    string). No `mode` logic in v2.0.0.
- `src/runtime/` — Meta/Shell-dependent units, each one responsibility:
  - `configWatcher.ts` — directory `Gio.FileMonitor` (`WATCH_MOVES`) on the config dir;
    debounced change callback (current `_startWatch` + `_scheduleApply`).
  - `desktopBackgrounds.ts` — apply per-monitor `Meta.Background` to
    `Main.layoutManager._bgManagers`, load-then-assign with the epoch guard, re-assert on
    `changed` (current `_apply` / `_setBackgroundWhenLoaded` / `_makeBackground` /
    `_connectManagers`). `ZOOM = 5` stays hardcoded.
  - `overviewBackgrounds.ts` — overview manager discovery + override + per-manager
    `changed` re-assert (current `_overviewBgManagers` / `_applyOverview` /
    `_overrideManager` / overview signal wiring).
  - `loginSeed.ts` — gsettings `picture-uri`/`-dark` seed with the get!==set guard
    (current `_seedLoginBackground`).
- `src/extension.ts` — `enable()`/`disable()` construct the above from the resolved
  config path, connect `monitors-changed`, and tear down. Holds no business logic.
- `src/metadata.json` — unchanged except `version` 1 → 2; copied into `dist/` by the
  build.

Connector→file resolution and config parsing move into `src/lib/config.ts` so they are
unit-testable; everything touching `Meta`/`Main`/`Gio` stays in `src/runtime/` and is
verified by the parity checklist.

## Tooling & build

- `package.json` (devDependencies only): `esbuild`, `typescript`, `@girs/gnome-shell`
  (its dependency closure provides every gi:// namespace type this project imports —
  verified), `eslint` + `typescript-eslint` + `@eslint/js`. Scripts:
  - `build`: `esbuild src/extension.ts --bundle --format=esm --outdir=dist
    '--external:gi://*' '--external:resource://*'` then `cp src/metadata.json dist/`.
  - `check`: `tsc --noEmit`.
  - `lint`: `eslint src`.
  - `test`: esbuild the `src/lib/*.test.ts` to JS, then `node --test` (verified path).
- `tsconfig.json`: `strict`, `module`/`moduleResolution` for ESM, `types` from `@girs`,
  `noEmit` (esbuild does the transpile/bundle; tsc only checks).
- Output: `dist/extension.js` + `dist/metadata.json` form the loadable extension dir.

## Packaging

- `install.sh`: `npm ci && npm run build`, then symlink/copy `dist/` to
  `$XDG_DATA_HOME/gnome-shell/extensions/<uuid>` (replacing the current `src/` symlink).
- GitHub Actions (on tag `vX.Y.Z`): `npm ci && npm run build`, package `dist/` into
  `per-monitor-wallpaper-<version>.tar.gz` (top-level dir = extension contents), attach
  as a release asset.
- rpm-specs `gnome-shell-extension-per-monitor-wallpaper.spec`: `Source0` → the release
  asset; `%build` stays empty, **no `BuildRequires`**; `%install` copies the prebuilt JS
  + metadata; `Version` 1.0.2 → 2.0.0; update its README row; re-run the lint gate.

## Testing

- Automated (esbuild `src/lib/*.test.ts` → JS, `node --test`): `src/lib/config.ts` —
  `parseConfig` (valid / invalid / empty) and `fileForConnector` (monitor entry / falls
  back to default / missing → null).
- Build/stack green: `npm run build` produces `dist/extension.js` + `dist/metadata.json`;
  `npm run check` and `npm run lint` pass.
- Parity (manual, live session, the bar): install the built v2.0.0; verify per-monitor
  wallpapers, overview backgrounds, login seed, the rotated `HDMI-1` monitor, and live
  `config.json` reload all match 1.0.2. Compare against the running 1.0.2 first.

## Non-goals

- Honoring `mode`, the preview UI, fit-modes, Display Settings link — all v2.1.0.
- In-RPM transpile (the CI release asset keeps the RPM build-free).
